*docright.txt* Local AI documentation for Neovim

==============================================================================
CONTENTS                                                        *docright*

DocRight asks a local model for concise programming documentation about
selected code or the code near the cursor. Follow-up questions reuse the last
documented context.

==============================================================================
REQUIREMENTS                                             *docright-requirements*

Neovim 0.9+, curl, and Ollama running locally.

The default model is qwen2.5-coder:7b:
>
  ollama pull qwen2.5-coder:7b
<

==============================================================================
COMMANDS                                                     *docright-commands*

:DocRight
  Document the symbol and surrounding code near the cursor.

:DocRightAsk
  Ask a follow-up question about the last DocRight response.

:DocRightDebug
  Show the assembled prompt that DocRight will send for the current cursor
  context.

:DocRightConfigDebug
  Show the merged DocRight config and the resolved system prompt.

==============================================================================
MAPPINGS                                                     *docright-mappings*

Defaults:
>
  <leader>ad    document selection or cursor context
  <leader>aa    ask follow-up
<

Inside a DocRight response window, <leader>ad expands the current response line
or selected response text with more detail and a small title.

==============================================================================
DEBUG                                                           *docright-debug*

:DocRightDebug opens a scratch buffer containing the assembled prompt for the
current cursor context, including the resolved `system_prompt`.

==============================================================================
CONFIGURATION                                                   *docright-setup*

DocRight registers its default commands and mappings when Neovim loads the
plugin. Call setup() only when overriding defaults:
>
  vim.g.docright_opts = {
    mappings = {
      document = "<leader>ad",
      ask = "<leader>aa",
    },
  }
<

If you call setup() yourself, it wins over the auto-loaded defaults.

>
require("docright").setup({
  provider = "ollama",
  model = "qwen2.5-coder:7b",
  endpoint = "http://127.0.0.1:11434/api/generate",
  system_prompt = [[
You are DocRight, a concise programming documentation assistant.
Only answer questions about code and software engineering.
]],
  temperature = 0.2,
  num_predict = 180,
  keep_alive = "10m",
  max_context_lines = 50,
  max_response_lines = 8,
  window = {
    width = 64,
    height = 0.97,
    border = "rounded",
    position = "right",
  },
  mappings = {
    document = "<leader>ad",
    ask = "<leader>aa",
  },
})
<

==============================================================================
GUARDRAIL                                                   *docright-guardrail*

DocRight prompts the local model to answer only about programming languages,
source code, APIs, libraries, frameworks, developer tooling, compilers,
runtimes, and software engineering concepts.

Set `system_prompt` in your config to replace the built-in system prompt.
